> 智能扫描系统，精准识别已卸载应用、网盘挂载和 Docker 的残留目录及重复文件，一键安全清理 - Node.js 版。

## 功能特性

- **自动扫描** — 动态发现系统中所有 vol 卷（`/vol1` ~ `/vol10` 及 `@app*` 目录）
- **孤立目录检测** — 交叉比对已安装应用列表与实际文件目录，精准识别残留孤立目录
- **网盘残余** — 扫描 `/vol02/`，对比 `/etc/mountmgr/mount_info.json`，识别已卸载的网盘挂载点
- **多选删除** — 支持批量勾选，一键清理，附确认弹窗与路径预览
- **明暗主题** — 支持手动切换（深色/浅色）
- **Tab 切换** — 应用 / 网盘 / Docker / tmp / 去重 / 去空 / 系统清理 / 回收站 独立展示
- **Docker 管理** — 卷/网络/镜像 扫描在用/未用，一键批量删除
- **tmp 清理** — 扫描 `/tmp` 和 `/var/tmp` 下 24h+ 未访问的文件
- **文件去重** — SHA-256 哈希比对，4 worker 线程 + streaming hash 优化
- **音乐去重** — 基于元数据（ID3/FLAC）的智能重复音乐识别
- **空目录清理** — 扫描指定目录下的空目录，支持回收站/永久删除
- **目录浏览选择** — 内置文件选择器，支持浏览并选择扫描目录
- **系统清理** — 17 项扫描目标，三档风险过滤 + 一键推荐清理 + 清理前 manifest 备份
- **回收站批量清理** — 扫描所有 vol × 所有 UID 的 `.@#local/trash/`，按 mtime 30/90/365 天分级清理
- **启动警告弹窗** — 每次进入应用强制阅读 3 秒红底警示
- **响应式布局** — 适配桌面端与移动端
- **轻量架构** — 纯 Node.js ESM 实现，零 npm 依赖

## 安装方式

将 `fnclearup.fpk` 上传至 FnOS 应用中心安装。

> 需要 Node.js v24 运行时（自动依赖安装）
